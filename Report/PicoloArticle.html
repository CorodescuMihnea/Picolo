Abstract

This article aims to provide an overview of our implementation of the project requirements. The requirements can be found here. For the structuring of the development steps we used a simplified version of the normal Software Development Life Cycle. For a more detailed view of the development steps please see the section bellow. The version control system used was Github and the webpage host(for this article) was Github pages. We chose Github for its ease of use and extensive documentation. The interaction between the end-user(web sites) and our service is done via a REST API implemented using Node.js (part of the project requirement). For data persistence we used json as a traditional DBMS would be unnecessary considering we only need to store a small amount of information about our clients.

Development steps

  1. Planning

    We considered the planning part of this project to be the discussions between ourselves and the course assistant.
    Taking into account the fact that we received an already detailed project specification, no further work was needed here.

  2. Design

    The design was done using C4 diagrams as we considered UML to be overly complicated for this projects scope.

  3. Implementation

    To be updated on project delivery

  4. Testing

    Testing for this project was done in the form of unit tests and manual tests using mock implementations of client web applications.

Workflow

  1. User requests a custom service worker
  2. API delivers service worker as text
